# HTML 合同

- 必须输出单一 `report.html`
- 必须包含标题区
- 必须包含周期信息区，明确"本周 / 上周"的周次或日期范围
- 必须按固定顺序输出 3 个栏目：
  1. 核心数据趋势
  2. GMV 拆解
  3. 流量 拆解
- 每个栏目必须包含图表区或表格区
- 每个栏目必须包含栏目级结论区
- 必须包含数据说明区，列出使用的数据合同和关键口径
- 页面应支持基础交互浏览，至少包含锚点导航或栏目跳转

## 图表约定

- 所有图表（柱状图、折线图）使用内联 SVG，不依赖外部 CDN
- 数值坐标轴必须从 0 开始，禁止截断坐标轴
- 支持双轴折线图（左侧轴 + 右侧轴），用于同时展示 GMV 和曝光PV

## 表格约定

- 每个指标格同时展示"本周数值"和"上周环比"，格式：`数值 (环比%)`
- WoW 变化以上下箭头或颜色区分涨跌：上涨绿色，下跌红色
- 当数据为空时，显示"暂无数据"

## 数据合同

所有数据文件的"周五-周四周"列为周数（如 11、12、13、14），已是周粒度聚合结果，不需要按日聚合。

本 skill 依赖以下 6 个数据合同：

| 合同名 | 文件名模式 | 说明 |
|--------|-----------|------|
| 整体数据 | `整体数据*.csv/xlsx` | 全局汇总数据，含 GMV、买家数、转化率等 |
| 行业数据 | `行业数据*.csv/xlsx` | 按行业聚合，含后台一级类目名称 |
| 行业商品明细数据 | `行业商品明细数据*.csv/xlsx` | 商品级明细，含商品名称、店铺名称 |
| 内容渠道数据 | `内容渠道数据*.csv/xlsx` | 按内容渠道（商品/视频/动态/直播）聚合 |
| 商家销售明细数据 | `商家销售明细*.csv/xlsx` | 商家级销售明细，含店铺名称 |
| 资源位二级入口数据 | `资源位二级入口数据*.csv/xlsx` | 按资源位二级入口聚合 |
