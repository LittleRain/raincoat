# 周报生成 Prompt

基于工房业务标准化 spec 和声明的数据合同，生成最终交互式 HTML 周报。

Rules:

- 只使用标准化 spec 中声明过的指标、维度和数据合同
- 严格保持固定栏目顺序
- 对绘画+绘画周边与 VUP周边分别生成独立分析块
- 对环比变化未超过 5% 的指标，不要强行给出异常解释
- 商品归因只允许引用行业商品明细数据中的曝光、转化、买家、GMV 等已声明字段
- 重点商品影响结论不能只写“变化”，必须明确写出上涨、下跌或持平
- 渠道占比分析只允许基于内容渠道数据中的占比和绝对值变化
- 两张内容渠道分布表的字段固定为：GMV、买家数、下单转化，且每个指标都必须包含上周环比
- 缺少关键文件时阻塞输出，而不是猜测
- 页面中应包含标题、周期、导航、图表、结论、数据说明
