# 输入文件清单

将以下文件放入同一个输入目录，文件名允许带日期或周次后缀，但应满足对应 pattern：

## 必需文件（6个）

- `整体数据*` (.csv / .xlsx / .xls)
  - 字段：周五-周四周、商品曝光PV、PVCTR、商详UV、支付订单数、支付订单买家数、GMV（不减退款）、商详支付转化率-UV、GPM
  - 说明：全局汇总数据，每行为一周的聚合结果

- `行业数据*` (.csv / .xlsx / .xls)
  - 字段：周五-周四周、后台一级类目名称、商品曝光PV、PVCTR、商详UV、支付订单数、支付订单买家数、GMV（不减退款）、商详支付转化率-UV、GPM
  - 说明：按行业聚合；后台一级类目名称 = 行业

- `行业商品明细数据*` (.csv / .xlsx / .xls)
  - 字段：周五-周四周、商品名称、店铺名称、后台一级类目名称、商品曝光PV、PVCTR、商详UV、支付订单数、支付订单买家数、GMV（不减退款）、商详支付转化率-UV、GPM
  - 说明：商品级明细，含本周+上周数据

- `内容渠道数据*` (.csv / .xlsx / .xls)
  - 字段：周五-周四周、后台一级类目名称、内容类型、商品曝光PV、PVCTR、商详UV、支付订单数、支付订单买家数、GMV（不减退款）、商详支付转化率-UV、GPM
  - 说明：内容类型 = 内容渠道（商品/视频/动态/直播/其他）

- `商家销售明细*` (.csv / .xlsx / .xls)
  - 字段：周五-周四周、后台一级类目名称、店铺名称、商品曝光PV、PVCTR、商详UV、支付订单数、支付订单买家数、GMV（不减退款）、商详支付转化率-UV、GPM
  - 说明：商家级销售明细；注意文件名无"数据"后缀

- `资源位二级入口数据*` (.csv / .xlsx / .xls)
  - 字段：周五-周四周、后台一级类目名称、内容类型、资源位二级入口、商品曝光PV、PVCTR、商详UV、支付订单数、支付订单买家数、GMV（不减退款）、商详支付转化率-UV、GPM
  - 说明：按资源位二级入口聚合

## 通用约定

- 所有文件的"周五-周四周"列已是**周粒度聚合**，值为周数（如 11、12、13），无需再按日聚合
- 列名可能带尾随空格（如"周五-周四周 "），代码会自动处理
- `GMV（不减退款）` 在代码中 rename 为 `GMV`
- `商详支付转化率-UV` 在代码中 rename 为 `转化率`
