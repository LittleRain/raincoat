# Gongfang Weekly Report

这是一个面向工房交易周报场景的执行型 skill，用于读取指定 Excel/CSV 数据并生成交互式 HTML 周报。

这个 skill 假定业务口径、栏目结构、重点行业和分析规则已经内置在 skill 自身，不依赖仓库外部文档才能理解或运行。

## 它做什么

这个 skill 会：

- 读取一个输入目录中的 4 份数据文件
- 按“周五到周四”将日级数据聚合成周
- 输出核心数据趋势、重点行业分析、内容渠道分布、重点商品影响
- 生成一个可直接打开的 `report.html`

## 输入要求

把以下文件放入同一个输入目录。文件可以是 `.csv`、`.xlsx` 或 `.xls`，文件名允许带日期或周次后缀，但前缀应满足以下 pattern。

### 1. `整体数据*`

必需字段：

- `日期`
- `商品曝光PV`
- `PVCTR`
- `商详UV`
- `支付订单数`
- `支付订单买家数`
- `GMV（不减退款）`
- `商详支付转化率-UV`
- `GPM`

用途：

- 核心数据趋势
- 周级 GMV、买家数、下单转化率、GPM 聚合

### 2. `行业数据*`

必需字段：

- `日期`
- `后台一级类目名称`
- `商品曝光PV`
- `PVCTR`
- `商详UV`
- `支付订单数`
- `支付订单买家数`
- `GMV（不减退款）`
- `商详支付转化率-UV`
- `GPM`

用途：

- 绘画+绘画周边 与 `VUP周边` 两个重点行业的周级分析

### 3. `行业商品明细数据*`

必需字段：

- `周` 或 `周五-周四周`
- `商品名称`
- `后台一级类目名称`
- `商品曝光PV`
- `PVCTR`
- `商详UV`
- `支付订单数`
- `支付订单买家数`
- `GMV（不减退款）`
- `商详支付转化率-UV`
- `GPM`

用途：

- 重点商品影响分析
- 对商品 GMV、买家数、商详UV 的上涨 / 下跌进行解释

### 4. `内容渠道数据*`

必需字段：

- `日期`
- `后台一级类目名称`
- `内容类型`
- `商品曝光PV`
- `PVCTR`
- `商详UV`
- `支付订单数`
- `支付订单买家数`
- `GMV（不减退款）`
- `商详支付转化率-UV`
- `GPM`

用途：

- 重点行业的内容渠道分布
- 渠道 GMV、买家数、下单转化及其上周环比

## 数据口径要求

- 至少覆盖两个完整对比周期，否则无法生成“本周 vs 上周”分析
- `日期` 字段应能被标准日期解析
- `后台一级类目名称` 中应至少包含 `绘画`、`绘画周边`、`VUP周边`
- `内容类型` 应能区分如 `商品`、`视频`、`动态`、`直播` 等渠道
- 若列名存在尾部空格或轻微别名，当前脚本会做有限兼容，但不要依赖过度模糊的文件格式

## 输出内容

运行成功后会生成：

- `report.html`
  - 交互式周报页面
  - 包含 SVG 图表、指标卡、重点行业区块、渠道分布表、重点商品影响结论
- `run.log`
  - 执行日志
  - 当前 / 上周周期
  - 输入输出路径

## 如何运行

```bash
bash skills/gongfang-weekly-report/scripts/run-report.sh \
  --input-dir /path/to/input \
  --output-dir /path/to/output \
  --run-id gongfang-weekly-2026w14
```

示例：

```bash
bash skills/gongfang-weekly-report/scripts/run-report.sh \
  --input-dir /Users/you/data/gongfang \
  --output-dir /Users/you/output/gongfang \
  --run-id gongfang-weekly-2026w14
```

## 最小使用步骤

1. 准备好 4 份输入文件
2. 确认字段齐全，且至少覆盖两个完整周期
3. 把文件放到同一个输入目录
4. 运行 `scripts/run-report.sh`
5. 打开输出目录中的 `report.html`

## 目录结构

- `SKILL.md`
  - agent 工作流说明
- `skill.json`
  - skill 元数据
- `assets/`
  - HTML 合同、栏目结构、提示词、校验清单
- `examples/`

- `scripts/`
  - 执行脚本和依赖声明
