# 产品策略

## 原始目标

你正在运营一个服务线下漫展行业的线上平台，平台同时有内容和交易。你希望打造一个主要服务自己的行业 Agent，完成：

- 每天从各个社交渠道了解行业信息。
- 把信息处理成结构化数据。
- 上传活动到系统里。
- 同步业务数据。
- 分析业务数据并给出动作建议。

## 思路调整

正确方向不是一开始做“全自动运营 Agent”，而是先做“运营副驾驶”。

原因：

- 漫展平台涉及交易、活动发布和用户信任，自动发布或自动改价的错误成本高。
- 早期最大价值不是“覆盖所有渠道”，而是缩短从行业信号到平台动作的时间。
- 你本人是主要使用者，人工审批本身就是高质量反馈数据。
- 没有业务结果回流，Agent 只会变成日报生成器。
- 当前没有历史样本，因此反馈采集本身就是第一阶段产品的一部分。

## 产品定位

这个 Agent 的 v0.1 定位：

```text
每天帮你发现行业机会，生成结构化活动草稿，指出业务异常，并给出可审批的运营动作。
```

不是：

- 自动经营平台。
- 自动发布所有活动。
- 自动改价。
- 自动投放预算。
- 自动替代商务判断。

## 北极星指标

主指标：

- 从发现活动到生成可审核活动草稿的时间。
- 活动草稿人工确认耗时。
- 有证据动作建议的采纳率。
- 采纳建议后的 GMV、转化率或内容效率变化。

辅助指标：

- 活动抽取准确率。
- 字典解析准确率。
- 重复活动误判率。
- 有害推荐数量。

## 第一版范围

v0.1 必须包含：

- `dictionary_sync`：同步省市区、场馆、场地和活动类型字典。
- `daily_intel`：每天采集两个优先信息源。
- `draft_events`：生成未发布活动草稿。
- `business_sync`：同步基础业务指标。
- `recommend_actions`：生成带证据的动作建议。
- 审批与反馈记录。

v0.1 暂不包含：

- 全渠道采集。
- 自动发布。
- 自动改价。
- 自动投放。
- 自动联系主办方发送非模板化消息。

## 成功标准

30 天内达到：

- 每天自动发现 >= 20 条相关行业信号。
- 活动识别 precision >= 80%。
- 活动草稿创建成功率 >= 95%。
- 人工确认一个活动草稿 <= 30 秒。
- 每天生成 >= 5 条可执行动作建议。
- 有害推荐数量 = 0。
